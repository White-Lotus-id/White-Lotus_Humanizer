# Section Templates — Chapter 3

Templates for all Chapter 3 sections of a Institutional thesis.
Thesis Anchor required before generating any Chapter 3 content.

---

## Thesis Anchor Requirement

Paste at start of session:
```
[THESIS ANCHOR]
Title:
Category: [Research / Development]
General Objective:
Scope (one sentence):
Key methodology:
```

---

## Research Design / Research Method

**Target**: 150-250 words

**Opening (exact):**
"This chapter presents the research design, data sources, data gathering
procedure, instrumentation, and statistical tools used in the study."

**Format**: Narrate in future tense.
**Soul note**: Soul only in rationale sentences: "The researchers chose this
approach because..."

**VOICE-A Methodology Phase template:**
```
[Phase N: Name.] [One-sentence overview].
[Since [reason], The researchers are [doing specific action]].
[In this case, [what is being used or built] because [justification]].
[Also, [additional step or component]].
[Because of that, [what this enables or produces]].
[Basically, [summary of phase goal and outcome]].
```

---

## Data Gathering Procedure

**Target**: 200-350 words
**Format**: RAD (Requirement, Analysis, Development) structure

**Phase structure:**
- Phase 1: Requirement Gathering
- Phase 2: Design / Development
- Phase 3: Testing

**Transitions**: "First," "Then," "After that," "Next," "Finally," "Also"
**Soul note**: Soul is in why each phase was structured the way it was.

---

## Instrumentation

**Target**: 200-350 words
**Rule**: Name every component exactly. No generic names.
  - YES: "ESP32 microcontroller"
  - NO: "a microcontroller"
  - YES: "PZEM-004T voltage sensor"
  - NO: "a voltage sensor"

**VOICE-A Hardware Description template:**
```
**[Component Name]**: [Short appositive description].
[It is used to [primary function]].
[We need this because [justification]].
[In this case, [specific operational detail]].
[Also, [secondary function or connection to other components]].
[Basically, [how it fits into the overall system]].
```

**VOICE-B Instrumentation template:**
```
S1: The [component] [is used to / serves as] [primary function].
S2: It plays a critical role in [secondary function].
S3: [Condition or specification] is required/essential to [achieve purpose].
```

**Soul note**: Soul is in knowing exactly what was used and why.

---

## Statistical Tools

**Target**: 100-200 words
**Format**: Each tool justified independently.
**Pattern**: "The researchers used [tool] to [purpose]."
**Soul note**: No tool listed without a stated purpose.
